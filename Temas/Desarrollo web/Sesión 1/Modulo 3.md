Un buen desarrollador documenta lo que hace. En lugar de usar Word, usamos **Markdown**, un lenguaje especial para escribir texto formateado que los editores de código y plataformas como GitHub entienden perfectamente.

1. En VS Code, haz clic derecho en el espacio vacío del explorador de archivos (a la izquierda) y crea un "Nuevo Archivo" (New File).

2. Llámalo exactamente `README.md` (con mayúsculas, la extensión `.md` significa Markdown).

3. Escribe lo siguiente usando estos símbolos especiales:

```markdown
# Mi Diario de Desarrollo Web

Este proyecto es el resultado de mi primera sesión aprendiendo desarrollo web profesional.

## Herramientas clave de mi entorno (Stack):
* **Astro:** El orquestador que crea las páginas rápidamente.
* **Tailwind CSS:** El sistema de diseño utilitario.
* **VS Code:** Mi editor de código principal.

## Mis primeros descubrimientos técnicos:
1. Comprendí la anatomía de HTML (`head` para el navegador, `body` para el humano).
2. Me di cuenta que la clase `flex` junto con `justify-center` ayuda muchísimo a centrar 
   elementos sin tener que calcular píxeles matemáticamente.

> "El código que no se documenta es código que se olvida en un mes." - Un programador sabio.
```

_Pro-tip:_ En VS Code, si tienes el `README.md` abierto, puedes presionar `Ctrl+K` soltar y luego presionar `V` (Win) o `Cmd+K` luego `V` (Mac) para abrir una vista previa al lado derecho y ver cómo se transforman tus símbolos en texto bonito.
